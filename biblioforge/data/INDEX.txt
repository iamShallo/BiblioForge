✅ CHECKLIST SETUP
═══════════════════════════════════════════════════════════════════════════

Prima di iniziare:
  ☐ SQL Server 2019+ installato e avviato
  ☐ SQL Server Management Studio (SSMS) installato
  ☐ Python 3.8+ con librerie installate (pip install -r requirements.txt)
  ☐ File books_cleaned.xlsx presente

Setup Database:
  ☐ Leggere QUICK_START.txt (5 min)
  ☐ Eseguire test_database_connection.py
  ☐ Se BiblioForge non esiste:
      ☐ Aprire database_schema.sql in SSMS
      ☐ Eseguire lo script SQL
      ☐ Verificare il test di nuovo

Importazione Dati:
  ☐ Verificare nome server in import_to_database.py
  ☐ Eseguire: python import_to_database.py
  ☐ Attendere completamento (5 min)

Verifica Finali:
  ☐ Eseguire: python database_queries.py
  ☐ Controllare statistiche nel report
  ☐ Verificare in SQL Management Studio: SELECT * FROM Libri;

🆘 ERRORI PIÙ COMUNI
═══════════════════════════════════════════════════════════════════════════

"Impossibile connettersi"
  ① Verificare SQL Server sia avviato
  ② Controllare nome server (in SSMS)
  ③ Verificare BiblioForge database esista
  Vedi: DATABASE_SETUP_GUIDE.md → Troubleshooting

"File not found"
  ① Verificare books_cleaned.xlsx esista
  ② Se no, eseguire: python clean_books.py
  Vedi: DATABASE_SETUP_GUIDE.md → Troubleshooting

"Database non trovato"
  ① Creare il database: database_schema.sql
  ② Eseguire in SQL Management Studio
  Vedi: QUICK_START.txt → STEP 2
